## Overview

The Airdrop Contract enables projects to distribute tokens to their communities efficiently and fairly. It supports multiple distribution models and handles large-scale airdrops with gas optimization for the Stellar network.

### Distribution Models

- **Equal Distribution**: Every recipient gets the same amount
- **Weighted Distribution**: Recipients get amounts based on their weight/score
- **Claimable Distribution**: Recipients claim tokens themselves (gas-free for sender)

### Key Features

- **Batch Operations**: Process thousands of recipients efficiently
- **Multiple Models**: Support for equal, weighted, and claimable airdrops
- **CSV Import Support**: Easy integration with spreadsheet data
- **Gas Optimization**: Minimized transaction costs on Stellar
- **Claim Management**: Track claimed vs unclaimed tokens
- **Deadline Support**: Set claim deadlines for claimable airdrops

## Distribution Models Explained

### Equal Distribution

Every recipient receives exactly the same amount:

```
Total Airdrop: 1,000,000 tokens
Recipients: 10,000 addresses  
Amount per recipient: 100 tokens each

Gas cost: Sender pays for all distributions
Completion: Immediate upon transaction confirmation
```

### Weighted Distribution  

Recipients receive amounts based on their weight or score:

```
Total Airdrop: 1,000,000 tokens
Recipient A: Weight 100 → Gets 100,000 tokens (10%)
Recipient B: Weight 50  → Gets 50,000 tokens (5%)  
Recipient C: Weight 25  → Gets 25,000 tokens (2.5%)
... (825,000 tokens to remaining recipients)

Total Weight: 1,000 points
Distribution: (recipient_weight / total_weight) * total_tokens
```

### Claimable Distribution

Recipients claim their allocation themselves:

```  
Total Airdrop: 1,000,000 tokens
Claim Period: 90 days
Gas Model: Recipients pay their own gas to claim

Benefits:
- Sender only pays gas for setup
- Recipients can claim when convenient  
- Unclaimed tokens can be recovered after deadline
```

## CSV-Based Airdrop Walkthrough

Here's a complete example of running an airdrop using CSV data:

### Step 1: Prepare Your CSV Data

Create a CSV file with recipient addresses and amounts:

```csv
address,amount,weight
GABC123...,100000000000,100
GDEF456...,75000000000,75  
GHIJ789...,50000000000,50
GKLM012...,25000000000,25
```

### Step 2: Process CSV Data

```typescript
import fs from 'fs'
import csv from 'csv-parser'

interface AirdropRecipient {
  address: string
  amount: string
  weight?: number
}

async function loadAirdropData(csvPath: string): Promise<AirdropRecipient[]> {
  const recipients: AirdropRecipient[] = []
  
  return new Promise((resolve, reject) => {
    fs.createReadStream(csvPath)
      .pipe(csv())
      .on('data', (row) => {
        recipients.push({
          address: row.address,
          amount: row.amount,
          weight: row.weight ? parseInt(row.weight) : undefined
        })
      })
      .on('end', () => resolve(recipients))
      .on('error', reject)
  })
}
```

```typescript
async function executeWeightedAirdrop(
  recipients: AirdropRecipient[],
  totalAmount: string
) {
  const weightedRecipients = recipients.map(r => ({
    address: r.address,
    weight: r.weight || 1

  const airdropId = await contract.call('create_weighted_airdrop', {
    token: TOKEN_ADDRESS,
    recipients: weightedRecipients,

  while (!completed) {
    const result = await contract.call('execute_airdrop', {
      airdrop_id: airdropId,
      batch_size: 100
    })
    completed = result.completed
    console.log(`Processed ${result.processed_count} recipients`)
  }
  
  return airdropId
}
```

## Merkle Tree Implementation for Claimable Airdrops

### Generating Merkle Proofs

```typescript
import { keccak256 } from 'js-sha3'

class MerkleTree {
  private leaves: Buffer[]
  private tree: Buffer[][]
  
  constructor(recipients: Array<{address: string, amount: string}>) {
    // Create leaf nodes from recipient data
    this.leaves = recipients.map(recipient => 
      Buffer.from(
        keccak256(
          Buffer.concat([
            Buffer.from(recipient.address, 'hex'),
            Buffer.from(recipient.amount)
          ])
        ), 'hex'
      )
    )
    
    this.tree = this.buildTree(this.leaves)
  }
  
  private buildTree(leaves: Buffer[]): Buffer[][] {
    const tree = [leaves]
    
    while (tree[tree.length - 1].length > 1) {
      const currentLevel = tree[tree.length - 1]
      const nextLevel: Buffer[] = []
      
      for (let i = 0; i < currentLevel.length; i += 2) {
        const left = currentLevel[i]
        const right = currentLevel[i + 1] || left
        
        const combined = Buffer.concat([left, right].sort(Buffer.compare))
        const hash = Buffer.from(keccak256(combined), 'hex')
        nextLevel.push(hash)
      }
      
      tree.push(nextLevel)
    }
    
    return tree
  }
  
  getRoot(): string {
    return '0x' + this.tree[this.tree.length - 1][0].toString('hex')
  }
  
  getProof(index: number): string[] {
    const proof: string[] = []
    
    for (let level = 0; level < this.tree.length - 1; level++) {
      const currentLevel = this.tree[level]
      const pairIndex = index % 2 === 0 ? index + 1 : index - 1
      
      if (pairIndex < currentLevel.length) {
        proof.push('0x' + currentLevel[pairIndex].toString('hex'))
      }
      
      index = Math.floor(index / 2)
    }
    
    return proof
  }
}
```

### Creating Claimable Airdrop

```typescript
async function createClaimableAirdrop(recipients: AirdropRecipient[]) {
  // Generate Merkle tree
  const merkleTree = new MerkleTree(recipients)
  const merkleRoot = merkleTree.getRoot()
  
  // Calculate total allocation
  const totalAmount = recipients
    .reduce((sum, r) => sum + BigInt(r.amount), 0n)
    .toString()
  
  // Set 90-day claim deadline
  const claimDeadline = Math.floor(Date.now() / 1000) + (90 * 24 * 60 * 60)
  
  const airdropId = await contract.call('create_claimable_airdrop', {
    token: TOKEN_ADDRESS,
    merkle_root: merkleRoot,
    total_amount: totalAmount,
    claim_deadline: claimDeadline
  })
  
  // Store proofs for later claiming
  const proofs = recipients.map((recipient, index) => ({
    address: recipient.address,
    amount: recipient.amount,
    proof: merkleTree.getProof(index)
  }))
  
  return { airdropId, proofs }
}
```

## Batch Distribution Gas Considerations

### Optimal Batch Sizing

Stellar has different gas considerations compared to Ethereum:

```typescript
// Stellar-optimized batch sizes
const BATCH_SIZES = {
  equal_distribution: 200,     // Can handle more equal distributions
  weighted_distribution: 150,  // Slightly fewer due to calculations
  merkle_claims: 1000         // Claims are individual, very efficient
}

async function executeLargeAirdrop(airdropId: number, totalRecipients: number) {
  const batchSize = BATCH_SIZES.weighted_distribution
  const totalBatches = Math.ceil(totalRecipients / batchSize)
  
  for (let batch = 0; batch < totalBatches; batch++) {
    const result = await contract.call('execute_airdrop', {
      airdrop_id: airdropId,
      batch_size: batchSize
    })
    
    console.log(`Batch ${batch + 1}/${totalBatches} completed`)
    console.log(`Processed: ${result.processed_count} recipients`)
    
    if (result.completed) {
      console.log('Airdrop execution completed!')
      break
    }
    
    // Small delay between batches to avoid rate limiting
    await new Promise(resolve => setTimeout(resolve, 1000))
  }
}
```

### Gas Cost Estimation

```typescript
function estimateAirdropCost(
  recipientCount: number, 
  distributionType: 'equal' | 'weighted' | 'claimable'
): number {
  const BASE_FEE = 100 // stroops
  const PER_RECIPIENT_FEE = {
    equal: 50,      // stroops per recipient
    weighted: 75,   // slightly higher for calculations  
    claimable: 10   // just setup cost, recipients pay to claim
  }
  
  const setupCost = BASE_FEE + (recipientCount * PER_RECIPIENT_FEE[distributionType])
  
  return setupCost // Total cost in stroops
}

// Example: 10,000 recipient weighted airdrop
const cost = estimateAirdropCost(10000, 'weighted')
console.log(`Estimated cost: ${cost / 1e7} XLM`) // ~0.075 XLM
```

## Security Considerations

<CalloutBox type="warning" title="Merkle Tree Security">
For claimable airdrops, ensure your Merkle tree implementation is secure and proofs are generated correctly. Invalid proofs will prevent legitimate claims.
</CalloutBox>

### Best Practices

1. **Validate Recipient Lists**: Remove duplicates and validate all addresses before creating airdrops
2. **Test on Testnet**: Always test large airdrops on testnet first
3. **Batch Execution**: Use appropriate batch sizes to avoid transaction failures
4. **Monitor Progress**: Set up monitoring for airdrop execution and claims
5. **Secure Merkle Data**: For claimable airdrops, securely store and serve proof data

### Access Control

- **Airdrop Creation**: Anyone with sufficient tokens can create airdrops
- **Airdrop Execution**: Only the creator can execute prepared airdrops  
- **Token Claims**: Only recipients with valid proofs can claim from claimable airdrops
- **Query Functions**: All query functions are public

## Troubleshooting

### Common Issues

**"InsufficientAllowance" Error**
- Approve the airdrop contract to spend your tokens before creating airdrops
- Ensure the approved amount covers the total airdrop allocation

**"InvalidProof" Error (Claimable Airdrops)**
- Verify the Merkle proof is generated correctly
- Ensure the recipient address and amount match the original tree data
- Check that the claim hasn't already been processed

**"ClaimDeadlinePassed" Error** 
- The claim period has expired for this airdrop
- Creators can recover unclaimed tokens after the deadline

**Batch Execution Failures**
- Reduce batch size if transactions are failing
- Ensure sufficient gas/fee budget for large batches
- Check network congestion and retry if needed

### Recovery Functions

```typescript
async function recoverUnclaimedTokens(airdropId: number) {

  }

}
```
