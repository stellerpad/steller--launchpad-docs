## Overview

The Vesting Contract enables you to create time-locked token distribution schedules. This is essential for projects that need to distribute tokens to teams, advisors, or investors over time to align incentives and prevent immediate dumping.

### Vesting Models Supported

- **Linear Vesting**: Tokens unlock gradually over time in a straight line
- **Cliff Vesting**: All tokens unlock at once after a waiting period  
- **Hybrid Vesting**: Combines cliff and linear (e.g., 25% cliff after 1 year, then linear over 3 years)

### Key Features

- **Flexible Schedules**: Support for any vesting timeline and structure
- **Multiple Beneficiaries**: Create vesting schedules for multiple addresses
- **Revocable Schedules**: Optional ability to revoke unvested tokens
- **Partial Claims**: Beneficiaries can claim available tokens at any time
- **Schedule Queries**: Check vesting progress and claimable amounts

## Vesting Models Explained

### Linear Vesting

Tokens unlock gradually over the vesting period:

```
Vesting Amount: 1,000,000 tokens
Start Date: January 1, 2024  
Duration: 24 months (730 days)
Daily Unlock: ~1,370 tokens per day

Timeline:
Day 0:    0 tokens claimable
Day 365:  500,000 tokens claimable (50%)
Day 730:  1,000,000 tokens claimable (100%)
```

### Cliff Vesting

All tokens unlock at once after a waiting period:

```
Vesting Amount: 1,000,000 tokens
Cliff Period: 12 months
Unlock Date: January 1, 2025

Timeline:  
Before Jan 1, 2025: 0 tokens claimable
After Jan 1, 2025:  1,000,000 tokens claimable
```

### Hybrid Vesting

Combines cliff and linear vesting:

```
Total Amount: 1,000,000 tokens
Cliff: 25% (250,000 tokens) after 12 months
Linear: 75% (750,000 tokens) over 36 months after cliff

Timeline:
Month 0:  0 tokens claimable
Month 12: 250,000 tokens claimable (cliff)  
Month 24: 500,000 tokens claimable (cliff + 50% linear)
Month 48: 1,000,000 tokens claimable (100%)
```

## Calculating Vesting Progress

### Manual Calculation

You can calculate vesting progress manually using these formulas:

```typescript
function calculateVestedAmount(schedule: VestingSchedule, currentTime: number): bigint {
  const { start_time, cliff_duration, vesting_duration, total_amount } = schedule
  
  // Before vesting starts
  if (currentTime < start_time) {
    return 0n
  }
  
  const elapsed = currentTime - start_time
  
  // During cliff period  
  if (elapsed < cliff_duration) {
    return 0n
  }
  
  // After vesting completes
  if (elapsed >= vesting_duration) {
    return total_amount
  }
  
  // Linear vesting calculation
  const vestingElapsed = elapsed - cliff_duration
  const linearDuration = vesting_duration - cliff_duration
  const vestedAmount = (total_amount * BigInt(vestingElapsed)) / BigInt(linearDuration)
  
  return vestedAmount
}

function calculateClaimableAmount(schedule: VestingSchedule, currentTime: number): bigint {
  const vestedAmount = calculateVestedAmount(schedule, currentTime)
  return vestedAmount - schedule.claimed_amount
}
```

## Integration Examples

### Complete Vesting Dashboard

```typescript
import { Contract } from '@stellar/stellar-sdk'

class VestingDashboard {
  constructor(private contract: Contract) {}
  
  // Get all schedules for a user
  async getUserSchedules(userAddress: string) {
    const scheduleIds = await this.contract.call(
      'get_schedules_for_beneficiary',
      { beneficiary: userAddress }
    )
    
    const schedules = await Promise.all(
      scheduleIds.map(id => 
        this.contract.call('get_vesting_schedule', { schedule_id: id })
      )
    )
    
    return schedules.map(schedule => ({
      ...schedule,
      claimable: this.contract.call('calculate_claimable_amount', {
        schedule_id: schedule.id
      })
    }))
  }
  
  // Claim all available tokens across all schedules
  async claimAllAvailable(userKeypair: Keypair) {
    const scheduleIds = await this.contract.call(
      'get_schedules_for_beneficiary', 
      { beneficiary: userKeypair.publicKey() }
    )
    
    const claims = []
    
    for (const scheduleId of scheduleIds) {
      const claimable = await this.contract.call(
        'calculate_claimable_amount',
        { schedule_id: scheduleId }
      )
      
      if (claimable > 0) {
        const tx = await this.contract.call('claim_tokens', {
          schedule_id: scheduleId
        })
        tx.sign(userKeypair)
        claims.push(tx)
      }
    }
    
    return claims
  }
}
```

## Security Considerations

<CalloutBox type="warning" title="Revocable Schedules">
Revocable vesting schedules give significant power to the creator. Beneficiaries should understand this risk, especially for team members who might be terminated.
</CalloutBox>

### Best Practices

1. **Use Non-Revocable for External Parties**: Investors and advisors should typically receive non-revocable schedules
2. **Secure Creator Keys**: The schedule creator can revoke schedules, so protect these private keys
3. **Verify Schedule Parameters**: Double-check all vesting parameters before creating schedules  
4. **Test Calculations**: Verify vesting calculations are correct for your specific use case
5. **Monitor Events**: Set up monitoring for claims and revocations

### Access Control

- **Schedule Creation**: Anyone with tokens and approvals can create schedules
- **Token Claims**: Only the beneficiary can claim their vested tokens  
- **Schedule Revocation**: Only the original creator can revoke (if revocable)
- **Query Functions**: All query functions are public

## Troubleshooting

### Common Issues

**"InsufficientAllowance" Error**
- The token creator needs to approve the vesting contract to spend tokens
- Use the token contract's `approve` function before creating vesting schedules

**"InvalidDuration" Error**
- Ensure vesting duration is greater than cliff duration  
- Both durations must be greater than 0

**"NotBeneficiary" Error**
- Only the designated beneficiary can claim tokens from their schedule
- Verify you're calling from the correct address

**"InsufficientVested" Error**
- Not enough time has passed for tokens to vest
- Use `calculate_claimable_amount` to check available tokens

### Common Calculations

**Converting Time Periods to Seconds**
```typescript
const SECOND = 1
const MINUTE = 60 * SECOND
const HOUR = 60 * MINUTE  
const DAY = 24 * HOUR
const WEEK = 7 * DAY
const MONTH = 30 * DAY // Approximation
const YEAR = 365 * DAY
```

**Converting Token Amounts**
```typescript
// Convert human-readable amount to stroops (7 decimals)
function toStroops(humanAmount: string): string {
  return (parseFloat(humanAmount) * 1e7).toString()
}

// Convert stroops back to human-readable  
function fromStroops(stroops: string): string {
  return (parseInt(stroops) / 1e7).toString()
}
```
