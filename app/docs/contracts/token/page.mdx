## Overview

The Token Contract is the core of the Stellar Launchpad ecosystem. It enables anyone to launch a token with built-in features for distribution, trading, and management. The contract follows Stellar's token standards while adding enhanced functionality for token launches.

### Key Features

- **Standard Compliance**: Full compatibility with Stellar token standards
- **Admin Controls**: Token admins can mint, burn, and freeze tokens
- **Metadata Management**: On-chain storage of token name, symbol, and description
- **Launch Integration**: Seamless integration with the Launchpad Registry
- **Security**: Built-in access controls and validation

### Use Cases

- **Project Tokens**: Launch utility tokens for dApps and protocols
- **Community Tokens**: Create tokens for DAOs and communities  
- **Reward Tokens**: Issue tokens for loyalty programs and incentives
- **Governance Tokens**: Enable decentralized governance with voting tokens
- **Stablecoins**: Issue asset-backed or algorithmic stablecoins

## Events

The token contract emits the following events for tracking and integration:

### Transfer Event
```rust
Transfer {
    from: Address,
    to: Address, 
    amount: u128
}
```
Emitted when tokens are transferred between addresses.

### Mint Event  
```rust
Mint {
    to: Address,
    amount: u128
}
```
Emitted when new tokens are minted.

### Burn Event
```rust
Burn {
    from: Address,
    amount: u128  
}
```
Emitted when tokens are burned.

### AdminChanged Event
```rust
AdminChanged {
    old_admin: Address,
    new_admin: Address
}
```
Emitted when admin privileges are transferred.

## Security Considerations

<CalloutBox type="warning" title="Admin Privileges">
The token admin has significant control including minting and burning tokens. Choose admin addresses carefully and consider using multi-sig wallets for production tokens.
</CalloutBox>

### Access Control

- **Admin Only**: `mint`, `burn`, `set_admin`
- **Authorized Users**: `transfer` (must be approved or own the tokens)
- **Public**: All read functions (`balance`, `name`, `symbol`, etc.)

### Best Practices

1. **Use Multi-Sig Admins**: For production tokens, use multi-signature wallets as admins
2. **Test Thoroughly**: Always test token operations on testnet first
3. **Validate Recipients**: Ensure transfer recipients are valid Stellar addresses
4. **Monitor Events**: Set up monitoring for mint/burn events to track supply changes
5. **Secure Key Management**: Keep admin private keys secure and backed up

### Known Limitations

- **No Pausability**: Once deployed, token transfers cannot be globally paused
- **Irreversible Burns**: Burned tokens cannot be recovered
- **Single Admin**: Only one admin address is supported (use multi-sig for shared control)
- **Fixed Decimals**: The number of decimals cannot be changed after deployment

## Integration Examples

### TypeScript Integration

```typescript
import { Contract, SorobanRpc, Keypair } from '@stellar/stellar-sdk'

// Connect to token contract
const contract = new Contract(TOKEN_CONTRACT_ADDRESS)
const server = new SorobanRpc.Server(RPC_URL)

// Check token balance
async function getBalance(userAddress: string): Promise<string> {
  const result = await contract.call(
    'balance',
    { address: userAddress }
  )
  return result.toString()
}

// Transfer tokens
async function transferTokens(
  fromKeypair: Keypair,
  toAddress: string, 
  amount: string
): Promise<string> {
  const tx = await contract.call(
    'transfer',
    {
      from: fromKeypair.publicKey(),
      to: toAddress,
      amount: amount
    }
  )
  
  tx.sign(fromKeypair)
  const result = await server.sendTransaction(tx)
  return result.hash
}
```

### Python Integration

```python
from stellar_sdk import Keypair, Server, TransactionBuilder
from stellar_sdk.contract import Contract

# Initialize contract
contract = Contract(TOKEN_CONTRACT_ADDRESS)
server = Server("https://soroban-testnet.stellar.org")

# Get token info
def get_token_info():
    name = contract.functions.name().call()
    symbol = contract.functions.symbol().call()
    decimals = contract.functions.decimals().call()
    total_supply = contract.functions.total_supply().call()
    
    return {
        'name': name,
        'symbol': symbol, 
        'decimals': decimals,
        'total_supply': total_supply
    }

# Mint tokens (admin only)
def mint_tokens(admin_keypair, to_address, amount):
    tx = contract.functions.mint(to_address, amount).build_transaction(
        source_account=admin_keypair.public_key,
        network_passphrase="Test SDF Network ; September 2015",
        base_fee=100
    )
    
    tx.sign(admin_keypair)
    response = server.submit_transaction(tx)
    return response['hash']
```

## Troubleshooting

### Common Errors

**"Unauthorized" Error**
- Ensure you're calling admin functions from the admin address
- Check that you have permission to transfer tokens from the source address

**"InsufficientBalance" Error**  
- Verify the source address has enough tokens for the transfer
- Remember amounts are in stroops (multiply by 10^decimals)

**"InvalidAmount" Error**
- Ensure amounts are greater than 0
- Check for integer overflow with very large amounts

**"AlreadyInitialized" Error**
- The token contract has already been initialized
- Deploy a fresh contract instance for a new token
